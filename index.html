3. En un contexto de la herramienta de control de versiones Git: definir 'working directory', 'index', 'local repo', 'remote repo', 'dirty working tree' y 'merge conflict'.

  1. Working Directory: es un single checkout de una versión de un proyecto. Estos archivos son sacados de la base de datos comprimida en el directorio de Git y se colocan en el disco para poder ser usados o modificados.
  2. Index: también conocido como staging area, es un archivo simple, generalmente contenido en el directorio del git, que guarda información acerca de lo que irá en el próximo commit.
  3. Local Repo:copia del repositorio git que el usuario posee en su máquina a través de git clone.
  4. Remote Repo:repositorio que almacena archivos en un servidor externo. Se debe hacer un push desde el repositorio local para que los cambios se vean reflejados en el remoto.
  5. Dirty Working Tree: Working tree en donde existen cambios al repositorio que no han sido commiteados.
  6. Merge Conflict:problema que ocurre cuando dos branches han sido modificados en la misma sección. Github no es capaz de hacer merge automático, y requiere que el usuario modifique a conveniencia.
